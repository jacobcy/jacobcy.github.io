## 📚 知识收集：Cubox + Raindrop

### Cubox：稍后阅读

**用途**：保存文章、视频、播客，稍后处理

**工作流程**：
1. 看到好内容 → 一键保存到Cubox
2. 每天固定时间（早/晚）阅读收集的内容
3. 有价值的 → 高亮+评论 → 同步到Obsidian
4. 读完后归档

**优势**：
- 跨平台（浏览器插件、手机App）
- 自动保存网页内容（防止链接失效）
- 支持全文搜索

### Raindrop：书签管理

**用途**：管理常用网站、工具、资源

**分类结构**：

## ⏰ 时间管理：日历块 + 番茄钟

### 日历块（Time Blocking）

**原则**：把一天分成几个大块，每块只做一类事

**我的日程安排**：

```
06:00-07:00  早起阅读 + 咖啡
07:00-08:00  运动/冥想
08:00-12:00  深度工作（编码/写作）
12:00-13:00  午餐 + 休息
13:00-17:00  会议/协作/杂事
17:00-18:00  学习/阅读
18:00-19:00  晚餐
19:00-21:00  自由时间（投资研究/兴趣）
21:00-22:00  复盘 + 明日计划
22:00-06:00  睡眠
```

**关键**：
- 早上做最重要的事（深度工作）
- 下午处理杂事（低精力时段）
- 晚上不工作，留给自己

### 番茄工作法

**工具**：Forest App / Pomofocus

**规则**：
- 25分钟专注 + 5分钟休息
- 每4个番茄钟，长休息15-30分钟
- 一个番茄钟内不切换任务

**适用场景**：
- 写代码
- 写博客
- 学习新技术

## 🤖 自动化工作流

### GitHub Actions：自动化部署

**场景**：博客自动部署

```yaml
name: Deploy Blog

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Setup Node
        uses: actions/setup-node@v3
        with:
          node-version: '18'
      
      - name: Install & Build
        run: |
          npm install
          npm run build
      
      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./public
```

## ⚠️ 工具链的陷阱

### 陷阱1：工具完美主义

**问题**：花更多时间配置工具，而不是使用工具

**解决**：
- 80/20法则：20%的功能满足80%的需求
- 先用起来，再逐步优化
- 不要频繁更换工具

### 陷阱2：过度自动化

**问题**：为了自动化而自动化，反而更复杂

**解决**：
- 手动做3次才考虑自动化
- 自动化的收益 > 维护成本

### 陷阱3：数据焦虑

**问题**：追踪太多数据，反而焦虑

**解决**：
- 只追踪最重要的3-5个指标
- 数据是参考，不是目标

## 🎯 给你的建议

如果你也想建立效率工具链，我的建议是：

### 第一步：确定核心需求（1周）

问自己：
- 我最常做哪些重复性工作？
- 我的信息是如何流动的？
- 我想提升什么方面？

### 第二步：选择基础工具（2-4周）

从简开始：
- 笔记：Obsidian（本地）或 Notion（在线）
- 任务：Notion 或 Todoist
- 日历：系统自带

### 第三步：建立工作流（1-2个月）

形成习惯：
- 每天早上看今日任务
- 固定时间处理收件箱
- 每周日复盘

### 第四步：逐步优化（持续）

根据使用情况调整：
- 添加自动化
- 优化模板
- 引入新工具
